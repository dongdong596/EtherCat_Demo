# AX58100 从上电到 OP 状态完整流程

> 硬件: AX58100 ESC + STM32F103VBT6 | 接口: SPI Mode 3, 4.5MHz | 主站: TwinCAT

---

## 完整流程：从上电到 OP

```
┌─────────────────────────────────────────────────────────────────┐
│  STM32 上电                                                     │
│    │                                                            │
│    │  ① 硬件初始化                                              │
│    ├─ HAL_Init()                     // CubeMX 生成的系统初始化  │
│    ├─ SystemClock_Config()           // HSE 8MHz × 9 = 72MHz   │
│    ├─ MX_GPIO_Init()                 // PA4=CS 输出高            │
│    ├─ MX_SPI1_Init()                 // SPI Mode3, 4.5MHz       │
│    │                                                            │
│    │  ② 认识 ESC                                                 │
│    ├─ AX58100_ReadESCInfo()          // 5 次 SPI 事务            │
│    │    块读 0x0000~0x0009 → type/revision/FMMU/SM/RAM         │
│    │    块读 0x0010~0x0015 → MAC                                │
│    │    单读 0x0140 → PDI Control                               │
│    │    单读 0x030D → PDI Error Counter                         │
│    │    单读 0x030E → PDI Error Code                            │
│    │    → 全部存入 g_escInfo                                    │
│    │                                                            │
│    │  ③ 设置初始状态                                             │
│    ├─ ECAT_Init()                    // 写 AL Status = Init     │
│    │    ESC_WriteRegister(0x0130, 0x01)  // bit3:0 = 1 (Init)  │
│    │    ESC_WriteRegister(0x0131, 0x00)  // 高字节 = 0          │
│    │                                                            │
│    ├─ CoE_Init()                     // 对象字典已静态初始化     │
│    │                                                            │
│    ├─ ESC_Watchdog_Config()          // 禁用看门狗               │
│    │    ESC_WriteRegister(0x0410, 0x0000)  // PDI 看门狗 = 0   │
│    │    ESC_WriteRegister(0x0420, 0x0000)  // SM 看门狗 = 0    │
│    │                                                            │
│    ▼                                                            │
│  ┌──────────────────────────────────────────────────────────┐   │
│  │  while(1) 主循环, 每 10ms:                                 │   │
│  │    ① ECAT_MainTask()        ← 状态机                       │   │
│  │    ② CoE_MainTask()         ← 邮箱 SDO 处理                │   │
│  │    ③ ECAT_ProcessDataExchange() ← 过程数据                 │   │
│  │    HAL_Delay(10)                                          │   │
│  └──────────────────────────────────────────────────────────┘   │
└─────────────────────────────────────────────────────────────────┘
```

## 状态切换全流程

### 上电 → Init

```
上电时 AX58100 硬件自身就是 Init 状态
ECAT_Init() 写 AL Status = 0x01 确认软件也认为自己是 Init

此时 TwinCAT 插网线：
  主站通过网线读 AL Status → 看到 0x01 = Init
  主站通过网线写 AL Control = 0x01 → "请保持在 Init"
  从站 STM32 通过 SPI 读到 AL Control = 0x01
  ECAT_MainTask() 发现 requestedState == m_currentState → 回 AL Status = 0x01
```

### Init → PreOp（两轮握手）

**第1轮 — ACK 试探：**

```
主站通过网线写 AL Control = 0x12
        │          │
        │          └─ bit3:0 = 0x02 (目标 PreOp)
        └──────────── bit4 = 1     (ACK! 叫你确认)

STM32 侧 ECAT_MainTask():
  ① ESC_ReadRegister(0x0120, &alCtrlLo) → 读到 0x12
  ② requestedState = 0x12 & 0x0F = 0x02  (PreOp)
  ③ ackBit         = 0x12 & 0x10 = 0x10  (有 ACK)
  ④ _ECAT_DoTransition(0x02, 0x10):
       IsValidTransition(Init, PreOp) → true ✓
       OnLeaveState(Init)           → 无操作
       OnEnterState(PreOp)          → ESC_SM_Init()  ← 配 SM0~SM3
       m_currentState = PreOp
       写 AL Status:
         statusLo = 0x02 | 0x10 = 0x12  ← Response 位 = ACK 位
         ESC_WriteRegister(0x0130, 0x12)     ← 回"我确认好了, 当前 PreOp"
         ESC_WriteRegister(0x0131, 0x00)
```

**第2轮 — 正式切：**

```
主站收到 Response=1 后：
主站通过网线写 AL Control = 0x02  (ACK=0, 正式切)

STM32 ECAT_MainTask():
  ① 读到 alCtrlLo = 0x02
  ② requestedState = 0x02  (PreOp)
  ③ ackBit = 0x00
  ④ requestedState == m_currentState(PreOp) → 走"状态没变但回应 ACK"分支
        statusLo = 0x02 | 0x00 = 0x02
        ESC_WriteRegister(0x0130, 0x02)  ← 回"当前 PreOp, 正式切好了"
```

**此时 ESC_SM_Init() 已执行：**

```
┌──────────────────────────────────────┐
│ ESC RAM 布局:                        │
│  0x1000 ~ 0x107F  SM0  128B  邮箱收 │
│  0x1080 ~ 0x10FF  SM1  128B  邮箱发 │
│  0x1100 ~ 0x111F  SM2  32B   过程收 │
│  0x1120 ~ 0x113F  SM3  32B   过程发 │
│                                      │
│ SM 寄存器已写:                        │
│  SM0: addr=0x1000, len=128, 邮箱模式  │
│  SM1: addr=0x1080, len=128, 邮箱模式  │
│  SM2: addr=0x1100, len=32,  缓冲模式  │
│  SM3: addr=0x1120, len=32,  缓冲模式  │
└──────────────────────────────────────┘
```

**进入 PreOp 后，TwinCAT 应该开始发 CoE SDO 请求了：**

```
CoE_MainTask() 每 10ms 循环:
  ① ESC_Mbx_RxFull() → ESC_ReadRegister(0x0805, &status)
     检查 SM0 状态寄存器 bit3 → 有数据吗？

  ┌─ 如果有(bit3=1):
  │  ② ESC_Mbx_Read(rxBuf, 128)
  │     ESC_ReadBlock(0x1000, rxBuf, 128)  ← 从 SM0 物理区读
  │     必须读到第 128 字节, ESC 才清 full 标志
  │
  │  ③ 解析: pMbxHdr = rxBuf
  │     if (type != 0x03) return;  ← 只要 CoE
  │
  │  ④ 解析 pCoEHdr = rxBuf + 6
  │     if (service != 0x02) return;  ← 只要 SDO 请求
  │
  │  ⑤ 解析 pSDO = rxBuf + 8
  │     根据 cmd 分发:
  │     ├─ 0x40 → SDO_HandleUpload(index, subindex)
  │     │          OD_Find() 查字典 → 打包 CoE 帧 → ESC_Mbx_Write() 发给 SM1
  │     └─ 0x20 → SDO_HandleDownload(pSDO)
  │                OD_Find() 查字典 → memcpy 写数据 → ESC_Mbx_Write() 回确认
  │     g_dbg_coe_procCnt++  ← 成功处理
  │
  │  ⑥ ESC_Mbx_Write(txBuf, ...)
  │     ESC_WriteBlock(0x1080, txbuf, 128)  ← 写到 SM1 物理区
  │     必须写满 128 字节, ESC 才置 SM1 的 full 位通知主站
  └─ 如果没有:
         return 0;  ← 无事发生

  ⚠️ 当前状态: 主站不发 SDO, 因为缺 ESI/XML 文件
     → g_dbg_coe_rxCnt 始终 = 0
     → CoE_MainTask() 每次快速返回
```

### PreOp → SafeOp

```
主站配完 CoE 对象后 (通过 SM0/SM1)，继续推状态:
主站通过网线写 AL Control = 0x14  (目标 SafeOp, ACK=1)

_ECAT_DoTransition(0x04, 0x10):
  OnLeaveState(PreOp)          → 无操作
  OnEnterState(SAFEOP)         → 目前是空函数 (TODO)
  写 AL Status = 0x14

第2轮: 主站写 0x04, 从站回 0x04 → 正式进入 SafeOp

此时:
  CoE 邮箱照常可用（主站可能继续配 FMMU）
  过程数据通道配好了但还没激活 ← 要 OP 才跑
```

### SafeOp → Op

```
主站通过网线写 AL Control = 0x18  (目标 Op, ACK=1)

_ECAT_DoTransition(0x08, 0x10):
  OnLeaveState(SAFEOP)          → 空
  OnEnterState(OP):
    memset(m_pdInput,  toggle, 32)   // 初始化输入区
    memset(m_pdOutput, 0,      32)   // 清输出区
  写 AL Status = 0x18

第2轮: 主站写 0x08, 从站回 0x08 → 正式进入 Op

从此刻起, ECAT_ProcessDataExchange() 每周期激活:
  if (m_currentState != OP) return;    ← 终于不直接返回了

  ESC_ReadOutputData(m_pdOutput, 32):
    ESC_SM_ReadConfig(2, &sm2)         ← 读 SM2 当前地址（可能被主站改过）
    ESC_ReadBlock(sm2.startAddr, m_pdOutput, 32)  ← 读主站发的过程数据

  m_pdInput[0]++                        ← 心跳计数

  ESC_WriteInputData(m_pdInput, 32):
    ESC_SM_ReadConfig(3, &sm3)         ← 读 SM3 当前地址
    ESC_WriteBlock(sm3.startAddr, m_pdInput, 32)  ← 写过程数据给主站
```

## 一图总结整个流程

```
上电
 │
 ├─ SPI 初始化, 读 ESC 身份信息
 ├─ ECAT_Init() → AL Status = Init
 ├─ ESC_Watchdog_Config() → 禁用看门狗
 │
 ▼ Init ──────────────────────────────────────
 │  什么都不做, 等主站
 │
 ▼ PreOp ── ESC_SM_Init() 配好 SM0~SM3 ──────
 │  SM0←主站问(CoE SDO) ─→ SM1→从站答
 │  主站读 0x1000/0x1008/0x1018...  知道"你是谁"
 │  主站写 SM/FMMU 配置               知道"数据在哪"
 │
 ▼ SafeOp ───────────────────────────────────
 │  配置验证完毕, FMMU 映射生效
 │  过程数据通道就绪, 但数据还不跑
 │
 ▼ Op ───────────────────────────────────────
    ECAT_ProcessDataExchange() 每 10ms:
      SM2 ← 主站写控制指令
      SM3 → 从站回状态数据
      全速运行!
```
