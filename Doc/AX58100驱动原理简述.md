# AX58100 驱动原理简述

## AX58100 到底是什么

AX58100 不是传统的"发命令干活的 SPI 外设"，它更像一个**共享内存交换机**。

```
┌─────────────────────────────────────────────────────────┐
│                                                         │
│   EtherCAT 主站                                          │
│   (TwinCAT / igh-ethercat)                              │
│      │                                                   │
│      │  以太网帧 (100Mbps)                                │
│      ▼                                                   │
│  ┌─────────────────────────────────────────────────┐    │
│  │                 AX58100                          │    │
│  │                                                  │    │
│  │  ┌──────────┐        ┌──────────────────┐       │    │
│  │  │  PHY×2   │◄──────►│    ESC Core       │       │    │
│  │  │          │        │                    │       │    │
│  │  └──────────┘        │  9KB 双端口 RAM    │       │    │
│  │                      │       ▲            │       │    │
│  │                      │       │            │       │    │
│  │                      │  ┌────┴────┐       │       │    │
│  │                      │  │ SPI PDI │       │       │    │
│  │                      │  └────┬────┘       │       │    │
│  │                      └───────┼────────────┘       │    │
│  └──────────────────────────────┼─────────────────────┘    │
│                                 │                          │
│                            PA4/5/6/7                       │
│                                 │                          │
│      ┌──────────────────────────┴──────────────────┐      │
│      │               STM32F103                     │      │
│      │  你的代码: 读写 RAM → 放到指定位置            │      │
│      │  AX58100:   主站来了 → 自动把 RAM 内容打包发走 │      │
│      │             主站发来数据 → 自动写入 RAM        │      │
│      └─────────────────────────────────────────────┘      │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

核心机制：AX58100 内部有一块 **9KB 双端口 RAM**。两个端口同时访问：

| 端口 | 谁在访问 | 干什么 |
|------|---------|------|
| ESC 端口 | EtherCAT 主站（通过网线） | 读/写过程数据，发邮箱命令 |
| PDI 端口 | 你的 STM32（通过 SPI） | 读/写过程数据，响应邮箱命令 |

两边**不需要互相同步**。你写完数据，主站下次发帧过来时自动带走。主站发过来的数据，自动出现在 RAM 里等你读。

---

## ESC 和 PDI 之间的通信

**是 AX58100 芯片内部硬件自动完成的，不需要你写任何代码。**

ESC（EtherCAT Slave Controller）和 PDI（Process Data Interface）是同一颗芯片里的两个模块，通过内部总线直接访问同一块双端口 RAM。你在 PDI 侧写一个字节到 0x1000，ESC 侧立刻就能看到。反过来也一样。

## 寄存器是"配置单"和"状态灯"

用快递柜比喻：

| 概念 | 比喻 |
|------|------|
| 双端口 RAM | 快递柜的格子 |
| SyncManager | "这个格子归谁用、要不要通知柜员" |
| FMMU | "主站说的门牌号对应哪个柜子号" |
| IRQ 寄存器 | "有人动过柜子"的提示灯 |
| AL Control/Status | 主站的指令单和你的回复单 |
| PDI Control | "柜员用 SPI 还是并口跟柜子说话" |

## 寄存器分类

### 1. 身份信息（0x0000~0x000F）— "这个柜子是谁"

```
0x0000 Type      = 0xC8    "我是 ASIX AX58100"
0x0001 Revision  = 0x00    "硬件版本 0"
0x0004 FMMU数量  = 0x08    "支持 8 个地址映射通道"
0x0005 SM数量    = 0x08    "支持 8 个数据通道"
0x0008 RAM大小   = 0x09    "有 9KB RAM"
0x0010 MAC地址             "我的网卡地址"
```

主站扫网时先读这些，知道挂了个什么设备。

### 2. SyncManager（0x0800~0x087F）— "数据放哪个格子里"

SM 是核心。每个 SM 管一块 RAM 区域，告诉 ESC 硬件：

```
SM 结构 (每 SM 16 字节, 共 8 个 SM):

偏移 0-1: 物理起始地址    → "从 RAM 的哪个地址开始"
偏移 2-3: 长度            → "这块区域多大"
偏移 4:   控制            → "方向(输入/输出)、模式(缓冲/邮箱)"
偏移 5:   状态            → "当前状态"
偏移 6:   激活            → "是否启用"
偏移 7:   PDI控制         → "PDI 侧的控制选项"
```

**SM 的作用就是告诉硬件：**

> "主站把数据写到 RAM 0x1000~0x10FF 时，给 STM32 发个中断"
> "STM32 往 RAM 0x1100~0x11FF 写完数据后，下一帧自动打包给主站"

没有 SM 配置，主站和 STM32 都对同一块 RAM 乱写，数据全乱。

### 3. FMMU（0x0600~0x06FF）— "主站那边的地址怎么映射到柜子这边"

主站用逻辑地址访问从站，FMMU 把逻辑地址翻译成物理 RAM 地址：

```
主站发: "把 2 字节写到从站逻辑地址 0x0000"
  ↓ FMMU0 映射: 逻辑 0x0000 → 物理 0x1200
  ↓
物理 RAM 0x1200 被写入
  ↓ STM32 读 SM2 管理的 0x1200 位置
  ↓
拿到主站发的数据
```

### 4. 中断/状态（0x0200~0x03FF）— "有人动过柜子没有"

```
0x0220 AL Event Request    "主站切状态了 / SM 数据更新了 / 看门狗触发了"
0x030D PDI Error Counter   "SPI 通信错了多少次"
```

这些寄存器是 ESC 硬件自动更新的。每次你发 SPI 事务时，IRQ 状态自动出现在 MISO 上，不需要额外读。

### 5. PDI 配置（0x0140~0x015F）— "这个柜子用什么方式跟柜员说话"

```
0x0140 = 0x05    "柜员接口是 SPI"
0x0150[1:0]=11  "SPI 用 Mode 3"
0x0150[4]=0     "CS 低有效"
```

这些值从 EEPROM 加载，上电后只读。

### 6. 状态机控制（0x0120~0x013F）— "主站的指令和你的回应"

```
0x0120 AL Control    主站写: "你给我切到 SafeOp"
0x0130 AL Status     你写:   "我已经在 SafeOp 了"
0x0134 AL Status Code         "我因为在做某某事，暂时切不过去"
```

## EtherCAT 状态机

```
(上电) ──► Init ──► PreOp ──► SafeOp ──► Op
              │         │         │         │
              │  只能    │  邮箱    │  邮箱    │  邮箱
              │  配寄存器│  + 过程  │  + 过程  │  + 过程
              │         │  数据    │  数据    │  数据
              │         │         │  (只读)  │  (读写)
```

STM32 不控制状态机切换——**主站通过网线发命令来控制**。你只需要：

1. **读 AL Control 寄存器 (0x0120)** — 看主站让你切到什么状态
2. **写 AL Status 寄存器 (0x0130)** — 告诉主站"我准备好了"
3. **处理状态切换时主站发来的邮箱命令** — 配置 SM、配 FMMU、配 DC 时钟

## 数据流向

```
【输出数据: 主站 → STM32】
  主站发以太网帧 ──► ESC 自动写入 SM2 管理的 RAM 区 ──► STM32 读走

【输入数据: STM32 → 主站】
  STM32 写入 SM3 管理的 RAM 区 ──► ESC 在下一帧到来时自动打包 ──► 主站收到

【配置命令: 主站 → STM32】
  主站发 CoE 邮箱命令 ──► ESC 写入 SM0 管理的 RAM 区 ──► STM32 读走
  STM32 处理后，写响应到 SM1 ──► 下一帧主站取走
```

## 一个完整的数据流例子

```
主站想让 STM32 控制的一个 IO 口翻转:

1. 主站通过 TwinCAT 配好:
   - SM2: 指向 0x1200, 长度 32, 方向=主站→STM32 (过程数据输出)
   - SM3: 指向 0x1300, 长度 32, 方向=STM32→主站 (过程数据输入)
   - FMMU0: 逻辑地址 0x0000 → 物理 0x1200 (输出数据映射)
   - FMMU1: 逻辑地址 0x0020 → 物理 0x1300 (输入数据映射)

2. 切到 Op 状态后，每周期自动:
   
   主站以太网帧过来:
   ┌────────────────────────────────────────────────────┐
   │ 帧头 │ ... │ 逻辑 0x0000: 0x01 │ ... │ FCS │
   └────────────────────────────────────────────────────┘
                           │
                           ▼ FMMU0 翻译
   ┌────────────────────────────────────────────────────┐
   │              RAM 0x1200 = 0x01                      │
   └────────────────────────────────────────────────────┘
                           │
                           ▼ SM2 触发中断 → STM32
                           │
   STM32: ESC_ReadBlock(0x1200, buf, 32)  // 读数据
          if (buf[0] == 0x01) { 翻转IO }
          buf[0] = 0x02  // 回复
          ESC_WriteBlock(0x1300, buf, 32)  // 写回
                           │
                           ▼ SM3 管理, 下一帧自动打包
   ┌────────────────────────────────────────────────────┐
   │ 帧头 │ ... │ 逻辑 0x0020: 0x02 │ ... │ FCS │
   └────────────────────────────────────────────────────┘
                           │
                           ▼ 主站收到 IO 状态
```

## 你用 SPI 访问的地址空间

```
AX58100 地址空间 (通过 SCS_ESC 访问):

0x0000 ─┐
0x0001  │ ESC 信息寄存器     (只读: Type, Version, FMMU数量...)
  ...   │
0x0100 ─┤
0x0140  │ PDI 控制寄存器     (EEPROM加载: PDI类型, SPI模式...)
0x0150  │
0x0200 ─┤
0x0220  │ 中断/状态寄存器    (IRQ, Watchdog, PDI Error...)
0x0300  │
0x0400 ─┤
0x0800  │ SyncManager 配置   (8个SM, 各16字节)
0x0880  │
0x0900 ─┤
0x0980  │ FMMU 配置          (地址映射)
0x0A00 ─┤
0x0F00  │
0x1000 ─┤
  ...   │ 过程数据 RAM       (你随便读写的区域)
  ...   │                    (SM 和 FMMU 会映射到这里)
0x1FFF ─┘
  ...   AX58100 实际有 9KB RAM, 地址空间 0x1000-0x1FFF 可直接访问
```

## 开发路线

```
✅ 第1步: SPI 物理层通信              ← 已完成
✅ 第2步: 单寄存器/块读写 ESC 寄存器   ← 已完成
⬜ 第3步: 读 ESC 完整信息 (FMMU/SM/RAM 数量)
⬜ 第4步: 实现状态机 (轮询 0x0120, 写 0x0130)
⬜ 第5步: 配置 SyncManager (SM0~SM3)
⬜ 第6步: 实现 CoE 邮箱协议 (通过 SM0/SM1)
⬜ 第7步: 处理过程数据 (SM2/SM3 + FMMU)
⬜ 第8步: 编写 ESI/XML 从站描述文件
```

## 一句话总结

> **AX58100 是一块"共享内存板"。** 主站通过网线往里写数据、往外读数据，你通过 SPI 往里写数据、往外读数据。你的代码不直接和 EtherCAT 协议打交道——你只需要在正确的地址写入正确的值，AX58100 会自动完成以太网帧的拼包拆包。你要做的事情就是：**配好 SM 地址 → 响应状态机 → 在指定 RAM 位置读写过程数据 → 写 ESI 文件告诉主站你的布局。**
